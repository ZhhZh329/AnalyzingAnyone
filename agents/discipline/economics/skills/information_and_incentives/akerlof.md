# 信息与激励镜头：Akerlof

## 核心理论框架

该镜头关注 adverse selection、quality uncertainty 与 trust breakdown。当质量、真实意图或履约能力难以在事前验证时，交换关系会退化，优质参与者退出，低质行为者反而更容易留下。分析重点不是一般性的“不信任别人”，而是此人如何面对 hidden type 环境：是否建立 screening、warranty、reputation filters、试用期或准入门槛，还是反复在质量不透明的交换中做出高代价决策。

## 分析方法论

优先看 biography、interviews、documented decisions 与 collaboration / conflict patterns 中的筛选、试用、声誉担保、信任破裂、合作者质量波动和关系退出。重点判断此人是建立 screening 机制、收紧入口、提高信息披露、要求可验证信号，还是反复在 lemons environment 中过度自信地交易。若证据无法区分“看不清类型”与“单纯性格不合”，不要硬套该镜头。

## Construct 分析指南

- **较强构念**：`epistemic_style`、`conflict_strategy`
- **次强构念**：`control_orientation`、`relation_to_hierarchy_power`
- **常见 not_applicable**：`mission_vs_relational_cost`、`abstraction_formalization`、`metacognition`
- `relation_to_hierarchy_power` 只有在 evidence 显示此人通过 gatekeeping、准入控制、层级筛选或 access control 来应对 hidden type 时才适合分析；普通的强势地位不够。
- `mission_vs_relational_cost` 通常应保持弱或 `not_applicable`；只有在此人反复为了维持质量筛选、排除低质量合作者而持续牺牲关系网络时才适合上调。

## 典型发现模式

- 因不信任质量披露而收紧合作边界，导致高质量伙伴也被排斥
- 在筛选不足的环境里反复遭遇“劣币驱逐良币”式合作退化
- 通过提高押注、准入门槛或可验证信息来稳定交换关系
- 把所有失败都归因于坏人，而忽略信息结构本身的逆向选择问题

## 局限、常见误读与谨慎推断

- **局限**：该镜头擅长解释 hidden type、质量不透明与筛选失败，不擅长单独解释广泛的人际不信任、价值冲突或深层心理动机。
- **常见误读**：不要把任何合作破裂都写成 lemons problem。关键是失败是否来自事前质量难验证、类型不透明或 screening 不足，而不是单纯的立场冲突。
- **不要从稀疏证据过度推断**：单次背叛、单个失败合作、单句“我不信任他们”，不足以推出稳定的 adverse-selection worldview。需要 repeated evidence 显示其持续面对并回应 hidden-type 风险。
- **不要过度分配 hierarchy / relationship constructs**：若没有明确的 gatekeeping、准入控制或为了保质而牺牲关系的 repeated behavior，`relation_to_hierarchy_power` 与 `mission_vs_relational_cost` 应保持弱或 `not_applicable`。
