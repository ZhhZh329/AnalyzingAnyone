# 数学分析分析框架 (Mathematical Analysis Framework)

## 1. 核心理论框架

数学分析强调极限、连续性、收敛、局部到整体的严密过渡，以及通过逐步论证建立可靠结论。用于人物分析时，它不是在评估其能否做微积分，而是在判断此人是否偏好严密推导、连续展开和层层收束的论证风格，还是更依赖启发式跳跃与快速近似。该框架对应“证明偏好 vs 启发式偏好”的关键一端，也特别适合分析其是否重视过程合法性而非只盯结果正确。

高数学分析倾向者通常不满足于“感觉上应该对”，他们更重视每一步如何从前一步合法推出，以及局部判断何以汇聚成整体结论；低此倾向者则更重可用性、速度和经验性跳跃，在现实场景中也可能非常有效。这个框架尤其适合分析一个人的严密性偏好、过程耐心、连续推理能力，以及其是否会用“严格”掩饰拖延。

## 2. 锚点人物与流派

- **Cauchy**：强调严密化，可用于观察其是否在意推导过程的合法性。
- **Weierstrass**：代表反直觉但精确的分析传统，适合分析其是否愿意牺牲直觉换取严格性。
- **Lebesgue**：体现“重建基础定义以获得更强表达力”，可用于判断其是否会为严密性重塑框架。
- **Hardy analysis tradition**：强调证明美学与严格性，可用于观察其是否偏好被证明的可靠感。

## 3. 分析方法论

1. **检查连续推理结构**：观察此人的论证是否具有明显的连续展开结构，而不是大跨度跳结论。

2. **分析局部到整体的过渡**：判断其是否区分局部成立与整体成立、短期有效与长期稳定。

3. **评估严密性容忍阈值**：考察其对“差不多”“大概是这样”的容忍度，以及是否会主动补足推理缺口。

4. **观察过程意识**：分析其是否重视变化轨迹、渐进过程和收敛路径，而不仅是起点和终点。

## 4. Construct 分析指南

- **agency_orientation**：分析型能动性表现为通过严密拆解与持续推进获得控制感，而非凭直觉猛冲。
- **epistemic_style**：核心看其是否偏好逐步论证、连续收敛和可验证推理链。
- **abstraction_formalization**：其形式化通常带有“把过程写清楚”的特征，而不只是搭一个静态模型。
- **control_orientation**：偏好掌握过程细节和过渡步骤，防止局部误差积累成全局失控。
- **mission_vs_relational_cost**：若过度追求严密，可能在实际合作中显得过慢、过细、压低节奏。
- **risk_posture**：这类人常厌恶未经验证的大跃迁，但一旦确认收敛路径，执行会更稳。
- **relation_to_hierarchy_power**：他们更相信证明链而不是地位背书，对“因为某专家说过”往往不满足。
- **conflict_strategy**：冲突时会追问哪一步不成立、哪一个局部推断不能推出整体结论。
- **time_horizon**：重视渐近演化与收敛过程，因此通常具备较强的过程耐心和长期推演能力。
- **rebuild_over_patch**：若发现基础论证有断点，往往倾向回到前面重推，而不是硬接后续步骤。
- **metacognition**：关键看其是否知道自己何时在追求真实严密，何时只是用严密姿态拖延决策。

## 5. 典型发现模式

- 强数学分析倾向者常在表达中层层推进，愿意为“这一步为什么成立”花大量时间。
- 他们通常善于看长期收敛、局部扰动和连续变化，但可能对快速试错容忍度较低。
- 弱此倾向者更偏经验近似和启发式跃迁，效率可能很高，但稳定性与可检验性较弱。
- 过强分析倾向可能导致完美主义、推进缓慢，甚至把需要行动的场景拖成无限证明。
