# 数理逻辑分析框架 (Mathematical Logic Framework)

## 1. 核心理论框架

数理逻辑关注命题、定义、推演规则与形式系统之间的关系。它分析的不是“一个观点是否动人”，而是这个观点是否定义清晰、前提明确、推导一致、结论可判定。将其用于人物分析时，核心问题是：此人是否倾向于把模糊议题压缩成可以判真假的结构化命题，并在论证中持续维护概念边界与推理一致性，是否习惯通过澄清定义、区分前提与结论、识别必要条件与充分条件来组织自己的思考。

从数理逻辑视角看，人物差异常体现在概念纪律和形式化冲动上。高数理逻辑倾向者通常重视定义先行、对歧义和偷换概念高度敏感，倾向于把讨论拉回规则和结构层面；低此倾向者则更依赖直觉、叙事与语境弹性。这个框架尤其适合分析一个人的形式化倾向、概念边界意识、论证一致性，以及其是否习惯把现实问题转写成较稳定的规则系统。

## 2. 锚点人物与流派

- **Frege**：强调概念清晰性与逻辑表达的精确性，可用于判断此人是否有“先把话说准再谈观点”的倾向。
- **Russell**：重视命题分解与逻辑消歧，可用于分析此人是否善于拆解复杂叙述中的混淆与歧义。
- **Hilbert**：代表形式主义传统，可用于观察此人是否偏好把问题转成符号规则和操作系统。
- **Gödel**：提醒形式系统存在内在边界，可用于分析此人是否既追求形式化，又意识到形式化的局限。

## 3. 分析方法论

1. **检查定义意识**：观察此人是否经常先定义术语、澄清前提、限定适用范围，再展开判断。

2. **分析推理链条**：考察其能否区分事实描述、价值判断、推论关系与边界条件，是否频繁依赖未说明的隐含前提。

3. **观察异议处理方式**：判断其面对分歧时，是倾向于回到定义和规则层面澄清，还是迅速转向修辞、立场或权威背书。

4. **评估形式化边界感**：分析其是否知道哪些问题可以被清晰形式化，哪些问题只能局部形式化，避免把一切都硬压成规则系统。

## 4. Construct 分析指南

- **agency_orientation**：逻辑框架下的高能动性常表现为争夺“定义权”和“问题表述权”，此人是否通过重写问题结构来改变局面。
- **epistemic_style**：重点判断其是否偏好可形式化、可判定、可一致检验的知识形态，而非模糊直觉或权威认同。
- **abstraction_formalization**：这是本框架的核心构念，观察其是否持续把现实问题转写为定义、命题、约束和规则系统。
- **control_orientation**：逻辑型控制常体现为对概念边界、讨论前提和论证流程的掌控，而非对人际情绪的直接控制。
- **mission_vs_relational_cost**：若其强烈追求逻辑一致，可能在讨论中牺牲礼貌、模糊空间或关系舒适度。
- **risk_posture**：关注其是否愿意在缺乏完全形式化保证的情况下行动，还是必须先把结构理清才肯推进。
- **relation_to_hierarchy_power**：逻辑倾向强的人常降低对正式权威的敬畏，更重视推理质量本身。
- **conflict_strategy**：是否习惯把冲突追溯到定义差异、隐藏前提或推理失配，而不是停留在表层分歧。
- **time_horizon**：形式化通常需要较高前期投入，观察其是否愿意为了后续清晰与可复用性承受初期成本。
- **rebuild_over_patch**：当概念结构存在根本歧义时，此人是否倾向于重写定义体系而非局部修补。
- **metacognition**：关键看其是否能反身性地检视自身论证中的隐含前提、偷换概念和形式边界。

## 5. 典型发现模式

- 强数理逻辑倾向者通常在表达中频繁下定义、限定条件、纠正他人用词，并对概念混淆明显不耐烦。
- 这类人往往擅长把含混争论转成结构化分歧，但可能低估现实中那些无法被完全形式化的人际和制度因素。
- 弱数理逻辑倾向者并不一定缺乏洞见，但其表达更依赖直觉、案例和叙事，不稳定维护概念边界。
- 过强形式主义者可能把“表达精确”误当成“现实已被把握”，从而高估形式系统对复杂现实的覆盖能力。
