# 动力系统分析框架 (Dynamical Systems Framework)

## 1. 核心理论框架

动力系统研究状态随时间演化的规律，包括稳定点、轨道、周期、吸引子、分岔、路径依赖和长期行为。用于人物分析时，它关注此人是否能以时间演化的眼光理解个体、组织和局势，而不是把系统只看成静态快照。该框架对应动态演化意识，也特别适合分析一个人是否善于看见积累效应、阈值跨越与长期轨迹，而不只盯住当下状态。

高动力系统倾向者通常很敏感于“现在这样，不代表以后还会这样”，他们关注状态如何迭代累积、何时会越过阈值、哪些扰动会放大成结构性变化，并会把注意力放在轨迹、吸引子和临界转折上；低此倾向者更容易做静态判断，把演化过程压缩成单点结论。这个框架尤其适合分析一个人的时间化理解能力、路径依赖意识和长期状态演化敏感度。

## 2. 锚点人物与流派

- **Poincaré**：动力系统早期关键人物，可用于分析其是否重视长期轨迹和非线性演化。
- **Lorenz**：混沌与敏感依赖代表，可用于判断其是否意识到小扰动的长期放大。
- **Strogatz**：现代动力系统传播者，适合分析其是否有相变、稳定性和分岔意识。
- **Nonlinear systems tradition**：可用于观察其是否避免过度线性化复杂现实。

## 3. 分析方法论

1. **识别时间化理解方式**：观察此人是否会把问题放入时间轴上理解，关注状态变化而非静态标签。

2. **检查路径依赖意识**：分析其是否识别积累效应、阈值突破和反馈导致的行为转变。

3. **评估阶段敏感性**：判断其是否能看出同样的输入在不同阶段会产生不同反应。

4. **观察吸引子与轨迹意识**：考察其是否理解系统常会被某些稳定模式反复拉回，以及改变轨迹为何困难。

## 4. Construct 分析指南

- **agency_orientation**：高动力系统能动性体现为不仅能行动，还能识别何时介入最能改变系统轨迹。
- **epistemic_style**：其认知偏好动态、过程化、演化式，而非一次性定性。
- **abstraction_formalization**：看其是否能将现实压成状态变量、演化规则和相位变化。
- **control_orientation**：其控制不只关注当下响应，还关注长期稳定、超调、振荡和吸引域切换。
- **mission_vs_relational_cost**：若过度轨迹导向，可能为了长期演化目标而容忍短期关系代价。
- **risk_posture**：对临界点、分岔和小扰动放大更敏感，风险理解也更具时间深度。
- **relation_to_hierarchy_power**：其可能更关注谁能改变系统轨道，而不是谁在某一时刻位置更高。
- **conflict_strategy**：冲突中可能分析互动是否正在进入恶性循环，并试图改变动力学结构。
- **time_horizon**：这是本框架核心，分析其是否习惯从长期演化而非短期状态理解人和局势。
- **rebuild_over_patch**：当系统吸引子本身有问题时，局部补丁难以生效，往往需要改演化机制。
- **metacognition**：关键看其是否意识到自己也被某些行为吸引子和历史轨迹持续塑造。

## 5. 典型发现模式

- 强动力系统倾向者常能看出“眼前问题只是长期轨迹的一个切片”。
- 他们擅长识别阈值、临界点和累积效应，适合分析组织演化、习惯形成和局势转折。
- 弱此倾向者更容易做静态归因，忽略同一系统随时间出现的相变和轨道漂移。
- 过强动力学视角可能导致什么都解释成“系统在演化”，从而削弱具体责任判断。
