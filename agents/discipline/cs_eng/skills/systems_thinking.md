# 系统思维分析框架（Systems Thinking）

## 1. 核心理论框架

系统思维将分析对象视为复杂自适应系统中的节点，而非孤立个体。核心假设：个体的行为模式、决策偏好和认知风格深受其所嵌入的反馈结构（feedback structures）影响。分析者需识别目标人物如何感知系统边界（system boundaries）、如何理解因果延迟（delays）、以及是否具备识别涌现现象（emergence）的能力。

## 2. 锚点人物与流派

- **Donella Meadows**：杠杆点理论（leverage points）——系统中存在少数高影响力干预点，分析目标人物是否倾向于寻找杠杆点而非表面症状；反馈回路（feedback loops）——正反馈与负反馈的识别能力。
- **Jay Forrester**：系统动力学（system dynamics）——强调库存-流量（stock-flow）思维，关注目标人物是否理解积累效应与非线性响应。
- **Peter Senge**：第五项修炼（5th discipline）——学习型组织视角，关注目标人物是否具备心智模型（mental models）反思能力和系统性愿景（shared vision）。

## 3. 分析方法论

从目标人物的决策叙事中提取以下信号：是否识别反馈环路、是否考虑二阶效应（second-order effects）、是否预判非预期后果（unintended consequences）、对系统延迟的耐受度、以及倾向于局部优化还是全局优化。通过其对失败案例的归因方式判断系统思维深度。

## 4. Construct 分析指南

- **agency_orientation**：是否认为个人行动能撬动系统变化，还是被系统结构所困。
- **epistemic_style**：偏好整体性理解（holistic）还是还原论式分解（reductionist）。
- **abstraction_formalization**：能否将具体问题抽象为反馈回路图或因果循环图（causal loop diagrams）。
- **control_orientation**：倾向于控制系统输入还是设计自调节机制（self-regulating mechanisms）。
- **mission_vs_relational_cost**：是否为系统级目标牺牲局部关系成本。
- **risk_posture**：对系统涌现风险的态度——拥抱不确定性还是试图消除之。
- **relation_to_hierarchy_power**：视层级为系统功能结构还是权力压制工具。
- **conflict_strategy**：是否将冲突视为系统张力的自然表达并利用之。
- **time_horizon**：是否关注长延迟效应，时间视野是否匹配系统动力学周期。
- **rebuild_over_patch**：面对系统性问题时，倾向于结构性重建还是局部修补。
- **metacognition**：系统思维者是否能识别自身的心智模型局限？是否会主动检验自己对系统边界和因果链的假设？高阶系统思维要求对"我用来理解系统的模型本身"进行反思。

## 5. 典型发现模式

高系统思维者常表现为：对短期KPI持怀疑态度、偏好设计机制而非直接指令、容忍短期混乱以换取长期结构改善、在归因时引用多因素交互而非单一原因。低系统思维者则倾向线性归因、忽视延迟效应、在局部优化中陷入"修补陷阱"（fixes that fail）。
