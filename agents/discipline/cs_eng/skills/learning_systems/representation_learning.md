# 表征学习分析框架 (Representation Learning)

## 1. 核心理论框架

表征学习关注的不是人知道多少事实，而是人如何把杂乱经验压缩成可迁移的内部表示。对人物分析而言，这个框架非常适合观察一个人是否擅长从噪声中提取稳定因子，是否能形成层次化概念，而不是陷在表面案例里打转。高水平的表征思维往往表现为：面对新问题时能迅速找到正确的特征空间，而不是把所有细节一股脑堆进脑中。

## 2. 锚点人物与流派

- **Yoshua Bengio**：系统阐释分层表征与可迁移特征的重要性。
- **Geoffrey Hinton**：通过深层网络推动表示学习成为现代 AI 核心。
- **Tomas Mikolov**：用嵌入空间展示语义结构可被压缩为向量关系。

## 3. 分析方法论

- 是否会主动寻找决定性的 latent factors，而不是平铺直叙地罗列事实。
- 是否能把多个案例压缩成一个更高层的模式。
- 面对复杂现象时，是否会区分信号与噪声、结构与表象。
- 是否能够迁移既有认知模板到新领域，而不是每次从零开始。

## 4. Construct 分析指南

- **agency_orientation**：是否试图通过如何构造内在表征、概念压缩与特征提取方式主动改写问题结构，而不是只在既有条件下被动应对。
- **epistemic_style**：其判断更依赖关于如何构造内在表征、概念压缩与特征提取方式的实验、度量、模型与证据，还是经验直觉和少量轶事。
- **abstraction_formalization**：是否能把如何构造内在表征、概念压缩与特征提取方式抽象成变量、接口、状态、约束、目标函数或可验证规范。
- **control_orientation**：更倾向直接操纵如何构造内在表征、概念压缩与特征提取方式的每个环节，还是通过机制、反馈和边界条件实现间接控制。
- **mission_vs_relational_cost**：为了推进如何构造内在表征、概念压缩与特征提取方式相关目标，是否愿意承受协作摩擦、组织阻力与人际成本。
- **risk_posture**：面对如何构造内在表征、概念压缩与特征提取方式中的不确定性、失败模式与尾部风险，是激进试错、分层防御还是保守回避。
- **relation_to_hierarchy_power**：在涉及如何构造内在表征、概念压缩与特征提取方式的资源、接口、权限和责任边界时，如何理解权威、分工与控制权。
- **conflict_strategy**：当围绕如何构造内在表征、概念压缩与特征提取方式的定义、指标、方案或责任发生分歧时，是诉诸实验、形式化、谈判还是权力推动。
- **time_horizon**：在如何构造内在表征、概念压缩与特征提取方式上更追求短期可见收益，还是愿意为长期能力建设、平台化积累与可维护性投资。
- **rebuild_over_patch**：当如何构造内在表征、概念压缩与特征提取方式暴露出系统性缺陷时，是局部修补还是重写关键假设、接口和架构。
- **metacognition**：是否能反思自己在如何构造内在表征、概念压缩与特征提取方式上的常见盲点，例如迷信指标、忽视边界条件、误把历史成功当普遍规律。

## 5. 典型发现模式

- 表象堆积型：信息很多，但缺乏可迁移结构。
- 压缩过度型：追求统一解释，容易忽略关键局部差异。
- 层级建模式：能同时处理具体实例与抽象模板，认知迁移能力强。
- 特征漂移型：会不断更换解释框架，显示出表征不稳定。
