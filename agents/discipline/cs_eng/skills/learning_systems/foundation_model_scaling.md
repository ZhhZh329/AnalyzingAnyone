# 基础模型与规模化学习分析框架 (Foundation Models & Scaling)

## 1. 核心理论框架

基础模型思维的核心不是把单个任务做到极致，而是通过足够通用的训练目标、足够大的预训练语料和统一接口，让能力在规模上自然涌现。映射到人物分析时，这个框架用于观察一个人是否喜欢构建通用底座、是否相信“平台化积累”会带来非线性收益，以及其对 emergent behavior 的敏感度。

## 2. 锚点人物与流派

- **Ilya Sutskever**：推动大模型预训练与规模化范式成为主流。
- **Dario Amodei**：系统讨论 scaling laws 与能力边界。
- **Alec Radford**：展示统一大模型接口在多任务上的泛化潜力。

## 3. 分析方法论

- 是否偏好先建通用底座，再在上层派生应用。
- 是否对规模带来的相变与涌现保持敏感。
- 是否能容忍长期高投入，只为积累未来复利。
- 是否理解统一接口、通用表示和多任务复用的战略价值。

## 4. Construct 分析指南

- **agency_orientation**：是否试图通过通过规模、预训练、通用接口与能力涌现来构建通用系统主动改写问题结构，而不是只在既有条件下被动应对。
- **epistemic_style**：其判断更依赖关于通过规模、预训练、通用接口与能力涌现来构建通用系统的实验、度量、模型与证据，还是经验直觉和少量轶事。
- **abstraction_formalization**：是否能把通过规模、预训练、通用接口与能力涌现来构建通用系统抽象成变量、接口、状态、约束、目标函数或可验证规范。
- **control_orientation**：更倾向直接操纵通过规模、预训练、通用接口与能力涌现来构建通用系统的每个环节，还是通过机制、反馈和边界条件实现间接控制。
- **mission_vs_relational_cost**：为了推进通过规模、预训练、通用接口与能力涌现来构建通用系统相关目标，是否愿意承受协作摩擦、组织阻力与人际成本。
- **risk_posture**：面对通过规模、预训练、通用接口与能力涌现来构建通用系统中的不确定性、失败模式与尾部风险，是激进试错、分层防御还是保守回避。
- **relation_to_hierarchy_power**：在涉及通过规模、预训练、通用接口与能力涌现来构建通用系统的资源、接口、权限和责任边界时，如何理解权威、分工与控制权。
- **conflict_strategy**：当围绕通过规模、预训练、通用接口与能力涌现来构建通用系统的定义、指标、方案或责任发生分歧时，是诉诸实验、形式化、谈判还是权力推动。
- **time_horizon**：在通过规模、预训练、通用接口与能力涌现来构建通用系统上更追求短期可见收益，还是愿意为长期能力建设、平台化积累与可维护性投资。
- **rebuild_over_patch**：当通过规模、预训练、通用接口与能力涌现来构建通用系统暴露出系统性缺陷时，是局部修补还是重写关键假设、接口和架构。
- **metacognition**：是否能反思自己在通过规模、预训练、通用接口与能力涌现来构建通用系统上的常见盲点，例如迷信指标、忽视边界条件、误把历史成功当普遍规律。

## 5. 典型发现模式

- 平台押注型：愿意忍受短期低效率，换取长期规模复利。
- 任务导向型：更喜欢针对性解法，难以接受通用底座的前期成本。
- 规模迷信型：相信“更大一定更好”，容易忽视数据质量与机制缺口。
- 涌现敏感型：会主动寻找能力相变点，并围绕这些拐点重构路线图。
