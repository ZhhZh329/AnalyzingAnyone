# 语义与规格分析框架 (Semantics & Specification)

## 1. 核心理论框架

语义与规格的核心是把模糊期待变成可共享、可验证、可争论的定义。用于人物分析时，它能区分一种人是靠暧昧共识推进事情，还是会主动澄清“到底什么算完成、什么算正确、什么算违背承诺”。

## 2. 锚点人物与流派

- **Dana Scott**：程序语义与形式化定义的重要人物。
- **Leslie Lamport**：把系统规格写成可推理对象的典范。
- **Bertrand Meyer**：契约式设计与规格思维的重要倡导者。

## 3. 分析方法论

- 是否对词义、接口承诺和成功标准高度敏感。
- 是否会在争议出现前先定义语义边界。
- 是否能把隐含预期显式化。
- 是否理解规格既是沟通工具，也是控制风险的手段。

## 4. Construct 分析指南

- **agency_orientation**：是否试图通过明确系统究竟意味着什么、保证行为与承诺一致主动改写问题结构，而不是只在既有条件下被动应对。
- **epistemic_style**：其判断更依赖关于明确系统究竟意味着什么、保证行为与承诺一致的实验、度量、模型与证据，还是经验直觉和少量轶事。
- **abstraction_formalization**：是否能把明确系统究竟意味着什么、保证行为与承诺一致抽象成变量、接口、状态、约束、目标函数或可验证规范。
- **control_orientation**：更倾向直接操纵明确系统究竟意味着什么、保证行为与承诺一致的每个环节，还是通过机制、反馈和边界条件实现间接控制。
- **mission_vs_relational_cost**：为了推进明确系统究竟意味着什么、保证行为与承诺一致相关目标，是否愿意承受协作摩擦、组织阻力与人际成本。
- **risk_posture**：面对明确系统究竟意味着什么、保证行为与承诺一致中的不确定性、失败模式与尾部风险，是激进试错、分层防御还是保守回避。
- **relation_to_hierarchy_power**：在涉及明确系统究竟意味着什么、保证行为与承诺一致的资源、接口、权限和责任边界时，如何理解权威、分工与控制权。
- **conflict_strategy**：当围绕明确系统究竟意味着什么、保证行为与承诺一致的定义、指标、方案或责任发生分歧时，是诉诸实验、形式化、谈判还是权力推动。
- **time_horizon**：在明确系统究竟意味着什么、保证行为与承诺一致上更追求短期可见收益，还是愿意为长期能力建设、平台化积累与可维护性投资。
- **rebuild_over_patch**：当明确系统究竟意味着什么、保证行为与承诺一致暴露出系统性缺陷时，是局部修补还是重写关键假设、接口和架构。
- **metacognition**：是否能反思自己在明确系统究竟意味着什么、保证行为与承诺一致上的常见盲点，例如迷信指标、忽视边界条件、误把历史成功当普遍规律。

## 5. 典型发现模式

- 语义清道夫型：擅长消除模糊带来的系统误解。
- 含混推进型：靠关系和速度前进，但后期返工多。
- 规格先行型：先谈定义后谈执行，适合高风险协作。
- 文档空转型：规格写得很多，却与现实实践脱节。
