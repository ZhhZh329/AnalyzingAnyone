# 编译器与运行时模型分析框架 (Compiler & Runtime Models)

## 1. 核心理论框架

编译器思维的核心是 translation across layers：把抽象意图安全、高效地落到具体执行。用于人物分析时，它考察一个人是否擅长在愿景、规范、流程、执行之间做语义保真转换，是否理解“写得出来”和“跑得起来”之间的鸿沟。

## 2. 锚点人物与流派

- **Frances Allen**：优化编译与程序转换的先驱。
- **John Backus**：高级语言与编译系统的重要推动者。
- **Guy Steele**：语言设计与运行时抽象的重要实践者。

## 3. 分析方法论

- 是否能够把高层目标翻译成可执行流程。
- 是否理解中间表示、约束传播和执行时开销。
- 是否擅长在不同抽象层之间保持语义一致。
- 是否能看见“规范正确但运行时失败”的根源。

## 4. Construct 分析指南

- **agency_orientation**：是否试图通过把高层意图映射为可执行机制，并理解执行时语义主动改写问题结构，而不是只在既有条件下被动应对。
- **epistemic_style**：其判断更依赖关于把高层意图映射为可执行机制，并理解执行时语义的实验、度量、模型与证据，还是经验直觉和少量轶事。
- **abstraction_formalization**：是否能把把高层意图映射为可执行机制，并理解执行时语义抽象成变量、接口、状态、约束、目标函数或可验证规范。
- **control_orientation**：更倾向直接操纵把高层意图映射为可执行机制，并理解执行时语义的每个环节，还是通过机制、反馈和边界条件实现间接控制。
- **mission_vs_relational_cost**：为了推进把高层意图映射为可执行机制，并理解执行时语义相关目标，是否愿意承受协作摩擦、组织阻力与人际成本。
- **risk_posture**：面对把高层意图映射为可执行机制，并理解执行时语义中的不确定性、失败模式与尾部风险，是激进试错、分层防御还是保守回避。
- **relation_to_hierarchy_power**：在涉及把高层意图映射为可执行机制，并理解执行时语义的资源、接口、权限和责任边界时，如何理解权威、分工与控制权。
- **conflict_strategy**：当围绕把高层意图映射为可执行机制，并理解执行时语义的定义、指标、方案或责任发生分歧时，是诉诸实验、形式化、谈判还是权力推动。
- **time_horizon**：在把高层意图映射为可执行机制，并理解执行时语义上更追求短期可见收益，还是愿意为长期能力建设、平台化积累与可维护性投资。
- **rebuild_over_patch**：当把高层意图映射为可执行机制，并理解执行时语义暴露出系统性缺陷时，是局部修补还是重写关键假设、接口和架构。
- **metacognition**：是否能反思自己在把高层意图映射为可执行机制，并理解执行时语义上的常见盲点，例如迷信指标、忽视边界条件、误把历史成功当普遍规律。

## 5. 典型发现模式

- 翻译高手型：善于把愿景转成机制，落地能力强。
- 高层空转型：目标清晰，但无法下沉到执行层。
- 低层绑架型：过度沉迷实现细节，丢失原始意图。
- 层间对齐型：能在抽象与执行之间反复校准。
