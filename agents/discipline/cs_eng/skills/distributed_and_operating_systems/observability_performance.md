# 可观测性与性能分析框架 (Observability & Performance)

## 1. 核心理论框架

可观测性代表一种工程诚实：系统到底在做什么，不靠猜，而靠持续可见的信号。用于人物分析时，这个框架能识别一个人是否依赖直觉拍板，还是会主动建设仪表盘、反馈链和定位机制。

## 2. 锚点人物与流派

- **Brendan Gregg**：性能分析与可观测性方法论的重要传播者。
- **Rob Pike**：系统可见性与简洁调试文化的代表人物。
- **Charity Majors**：现代可观测性工程实践的重要声音。

## 3. 分析方法论

- 是否优先建立度量和诊断入口，而不是只凭主观感觉。
- 是否在性能下降时先找系统证据。
- 是否能区分 symptom 与 root cause。
- 是否愿意长期维护可观测性基础设施。

## 4. Construct 分析指南

- **agency_orientation**：是否试图通过通过指标、日志、追踪与剖析理解系统行为和性能退化主动改写问题结构，而不是只在既有条件下被动应对。
- **epistemic_style**：其判断更依赖关于通过指标、日志、追踪与剖析理解系统行为和性能退化的实验、度量、模型与证据，还是经验直觉和少量轶事。
- **abstraction_formalization**：是否能把通过指标、日志、追踪与剖析理解系统行为和性能退化抽象成变量、接口、状态、约束、目标函数或可验证规范。
- **control_orientation**：更倾向直接操纵通过指标、日志、追踪与剖析理解系统行为和性能退化的每个环节，还是通过机制、反馈和边界条件实现间接控制。
- **mission_vs_relational_cost**：为了推进通过指标、日志、追踪与剖析理解系统行为和性能退化相关目标，是否愿意承受协作摩擦、组织阻力与人际成本。
- **risk_posture**：面对通过指标、日志、追踪与剖析理解系统行为和性能退化中的不确定性、失败模式与尾部风险，是激进试错、分层防御还是保守回避。
- **relation_to_hierarchy_power**：在涉及通过指标、日志、追踪与剖析理解系统行为和性能退化的资源、接口、权限和责任边界时，如何理解权威、分工与控制权。
- **conflict_strategy**：当围绕通过指标、日志、追踪与剖析理解系统行为和性能退化的定义、指标、方案或责任发生分歧时，是诉诸实验、形式化、谈判还是权力推动。
- **time_horizon**：在通过指标、日志、追踪与剖析理解系统行为和性能退化上更追求短期可见收益，还是愿意为长期能力建设、平台化积累与可维护性投资。
- **rebuild_over_patch**：当通过指标、日志、追踪与剖析理解系统行为和性能退化暴露出系统性缺陷时，是局部修补还是重写关键假设、接口和架构。
- **metacognition**：是否能反思自己在通过指标、日志、追踪与剖析理解系统行为和性能退化上的常见盲点，例如迷信指标、忽视边界条件、误把历史成功当普遍规律。

## 5. 典型发现模式

- 黑箱指挥型：判断常靠权威感，错误定位效率低。
- 证据驱动型：先量测再决策，定位问题更稳。
- 指标崇拜型：过度迷恋 dashboard，忽视真实语境。
- 根因猎手型：擅长顺着信号链条找到真正瓶颈。
