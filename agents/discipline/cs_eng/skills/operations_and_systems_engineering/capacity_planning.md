# 容量规划分析框架 (Capacity Planning)

## 1. 核心理论框架

容量规划是介于悲观与乐观之间的工程判断：既不能为永远不会发生的峰值过度投资，也不能在真实增长来临时手忙脚乱。用于人物分析时，这个框架看的是一个人对增长曲线和缓冲空间的感觉。

## 2. 锚点人物与流派

- **John Little**：排队与容量关系的经典定律代表。
- **Jeff Dean**：大规模系统容量与扩展规划的重要实践者。
- **John Allspaw**：生产容量、事故复盘与运维可靠性文化的重要人物。

## 3. 分析方法论

- 是否会预测需求变化并预留能力。
- 是否理解峰值、平均值和突发流量的不同。
- 是否能在冗余成本与爆仓风险之间平衡。
- 是否把容量问题视为持续管理而非临时采购。

## 4. Construct 分析指南

- **agency_orientation**：是否试图通过需求预测、弹性准备、资源冗余与增长节奏管理主动改写问题结构，而不是只在既有条件下被动应对。
- **epistemic_style**：其判断更依赖关于需求预测、弹性准备、资源冗余与增长节奏管理的实验、度量、模型与证据，还是经验直觉和少量轶事。
- **abstraction_formalization**：是否能把需求预测、弹性准备、资源冗余与增长节奏管理抽象成变量、接口、状态、约束、目标函数或可验证规范。
- **control_orientation**：更倾向直接操纵需求预测、弹性准备、资源冗余与增长节奏管理的每个环节，还是通过机制、反馈和边界条件实现间接控制。
- **mission_vs_relational_cost**：为了推进需求预测、弹性准备、资源冗余与增长节奏管理相关目标，是否愿意承受协作摩擦、组织阻力与人际成本。
- **risk_posture**：面对需求预测、弹性准备、资源冗余与增长节奏管理中的不确定性、失败模式与尾部风险，是激进试错、分层防御还是保守回避。
- **relation_to_hierarchy_power**：在涉及需求预测、弹性准备、资源冗余与增长节奏管理的资源、接口、权限和责任边界时，如何理解权威、分工与控制权。
- **conflict_strategy**：当围绕需求预测、弹性准备、资源冗余与增长节奏管理的定义、指标、方案或责任发生分歧时，是诉诸实验、形式化、谈判还是权力推动。
- **time_horizon**：在需求预测、弹性准备、资源冗余与增长节奏管理上更追求短期可见收益，还是愿意为长期能力建设、平台化积累与可维护性投资。
- **rebuild_over_patch**：当需求预测、弹性准备、资源冗余与增长节奏管理暴露出系统性缺陷时，是局部修补还是重写关键假设、接口和架构。
- **metacognition**：是否能反思自己在需求预测、弹性准备、资源冗余与增长节奏管理上的常见盲点，例如迷信指标、忽视边界条件、误把历史成功当普遍规律。

## 5. 典型发现模式

- 临渴掘井型：总在容量见底时才行动。
- 前瞻规划型：对增长节奏和资源缓冲有明确策略。
- 冗余过度型：安全感强但成本过高。
- 弹性设计型：善于通过可伸缩机制降低容量焦虑。
