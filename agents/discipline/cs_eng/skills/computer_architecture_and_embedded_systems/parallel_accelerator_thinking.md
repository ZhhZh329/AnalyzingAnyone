# 并行与加速器思维分析框架 (Parallelism & Accelerator Thinking)

## 1. 核心理论框架

并行与加速器思维的本质是寻找可同时执行的结构，并识别哪些瓶颈适合通过专用化解决。用于人物分析时，它能识别一个人是否有“把慢问题拆成可并行快问题”的能力。

## 2. 锚点人物与流派

- **Gene Amdahl**：并行加速受限于串行瓶颈的经典洞见。
- **John Gustafson**：从规模扩展角度重新理解并行收益。
- **David Patterson**：以体系结构视角推动专用加速。

## 3. 分析方法论

- 是否天然寻找可并行部分与不可并行瓶颈。
- 是否知道什么时候值得专门化，什么时候通用更好。
- 是否对数据搬运成本保持敏感。
- 是否理解吞吐提升不等于端到端加速。

## 4. Construct 分析指南

- **agency_orientation**：是否试图通过利用并行结构、专用加速与数据局部性提升系统性能主动改写问题结构，而不是只在既有条件下被动应对。
- **epistemic_style**：其判断更依赖关于利用并行结构、专用加速与数据局部性提升系统性能的实验、度量、模型与证据，还是经验直觉和少量轶事。
- **abstraction_formalization**：是否能把利用并行结构、专用加速与数据局部性提升系统性能抽象成变量、接口、状态、约束、目标函数或可验证规范。
- **control_orientation**：更倾向直接操纵利用并行结构、专用加速与数据局部性提升系统性能的每个环节，还是通过机制、反馈和边界条件实现间接控制。
- **mission_vs_relational_cost**：为了推进利用并行结构、专用加速与数据局部性提升系统性能相关目标，是否愿意承受协作摩擦、组织阻力与人际成本。
- **risk_posture**：面对利用并行结构、专用加速与数据局部性提升系统性能中的不确定性、失败模式与尾部风险，是激进试错、分层防御还是保守回避。
- **relation_to_hierarchy_power**：在涉及利用并行结构、专用加速与数据局部性提升系统性能的资源、接口、权限和责任边界时，如何理解权威、分工与控制权。
- **conflict_strategy**：当围绕利用并行结构、专用加速与数据局部性提升系统性能的定义、指标、方案或责任发生分歧时，是诉诸实验、形式化、谈判还是权力推动。
- **time_horizon**：在利用并行结构、专用加速与数据局部性提升系统性能上更追求短期可见收益，还是愿意为长期能力建设、平台化积累与可维护性投资。
- **rebuild_over_patch**：当利用并行结构、专用加速与数据局部性提升系统性能暴露出系统性缺陷时，是局部修补还是重写关键假设、接口和架构。
- **metacognition**：是否能反思自己在利用并行结构、专用加速与数据局部性提升系统性能上的常见盲点，例如迷信指标、忽视边界条件、误把历史成功当普遍规律。

## 5. 典型发现模式

- 并行拆分型：擅长把大任务重构为可同时推进的子任务。
- 串行惯性型：思路清晰但吞吐受限。
- 专用化激进型：过快追求定制化，牺牲灵活性。
- 瓶颈清醒型：总能看见真正限制加速的那部分。
