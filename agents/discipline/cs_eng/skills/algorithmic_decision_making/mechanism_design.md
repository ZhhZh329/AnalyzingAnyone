# 机制设计分析框架 (Mechanism Design)

## 1. 核心理论框架

机制设计不是在既有规则下出牌，而是重新定义游戏规则本身。用于人物分析时，这个框架特别适合看一个人是否善于通过 incentive design、接口权限与评价制度去改变系统行为，而不是永远停留在个体说服层面。

## 2. 锚点人物与流派

- **Leonid Hurwicz**：机制设计理论奠基者。
- **Eric Maskin**：研究激励兼容与实现问题。
- **Alvin Roth**：把市场设计带入现实制度工程。

## 3. 分析方法论

- 是否擅长通过规则改变行为，而不是反复要求个人更努力。
- 是否意识到信息不对称会重塑系统结果。
- 是否能设计让个体理性与整体目标更一致的机制。
- 是否敏感于指标作弊、逆向激励和制度空子。

## 4. Construct 分析指南

- **agency_orientation**：是否试图通过通过规则、激励与信息结构设计群体行为主动改写问题结构，而不是只在既有条件下被动应对。
- **epistemic_style**：其判断更依赖关于通过规则、激励与信息结构设计群体行为的实验、度量、模型与证据，还是经验直觉和少量轶事。
- **abstraction_formalization**：是否能把通过规则、激励与信息结构设计群体行为抽象成变量、接口、状态、约束、目标函数或可验证规范。
- **control_orientation**：更倾向直接操纵通过规则、激励与信息结构设计群体行为的每个环节，还是通过机制、反馈和边界条件实现间接控制。
- **mission_vs_relational_cost**：为了推进通过规则、激励与信息结构设计群体行为相关目标，是否愿意承受协作摩擦、组织阻力与人际成本。
- **risk_posture**：面对通过规则、激励与信息结构设计群体行为中的不确定性、失败模式与尾部风险，是激进试错、分层防御还是保守回避。
- **relation_to_hierarchy_power**：在涉及通过规则、激励与信息结构设计群体行为的资源、接口、权限和责任边界时，如何理解权威、分工与控制权。
- **conflict_strategy**：当围绕通过规则、激励与信息结构设计群体行为的定义、指标、方案或责任发生分歧时，是诉诸实验、形式化、谈判还是权力推动。
- **time_horizon**：在通过规则、激励与信息结构设计群体行为上更追求短期可见收益，还是愿意为长期能力建设、平台化积累与可维护性投资。
- **rebuild_over_patch**：当通过规则、激励与信息结构设计群体行为暴露出系统性缺陷时，是局部修补还是重写关键假设、接口和架构。
- **metacognition**：是否能反思自己在通过规则、激励与信息结构设计群体行为上的常见盲点，例如迷信指标、忽视边界条件、误把历史成功当普遍规律。

## 5. 典型发现模式

- 规则塑造型：通过激励和约束重构组织行为，影响范围大。
- 道德说服型：更相信价值宣讲，较少动用机制工程。
- 制度天真型：忽视 Goodhart 式副作用，容易被指标反噬。
- 逆向利用型：敏锐理解制度漏洞，也可能把系统推向机会主义。
