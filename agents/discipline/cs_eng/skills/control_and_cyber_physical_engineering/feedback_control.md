# 反馈控制分析框架 (Feedback Control)

## 1. 核心理论框架

反馈控制的核心洞见是：好的系统不是一次设定后永远正确，而是持续感知偏差并不断校正。用于人物分析时，这个框架可识别一个人是否习惯建立闭环，而不是只做开环命令。

## 2. 锚点人物与流派

- **Norbert Wiener**：控制论奠基者。
- **Rudolf Kalman**：现代控制与状态估计的重要代表。
- **Karl Åström**：反馈控制工程实践的重要学者。

## 3. 分析方法论

- 是否主动测量偏差并调整行为。
- 是否理解延迟、超调与稳定性。
- 是否善于建立闭环而不是靠一次性指令。
- 是否能接受持续调参而非一次到位。

## 4. Construct 分析指南

- **agency_orientation**：是否试图通过通过测量误差、闭环调节和稳定机制维持目标状态主动改写问题结构，而不是只在既有条件下被动应对。
- **epistemic_style**：其判断更依赖关于通过测量误差、闭环调节和稳定机制维持目标状态的实验、度量、模型与证据，还是经验直觉和少量轶事。
- **abstraction_formalization**：是否能把通过测量误差、闭环调节和稳定机制维持目标状态抽象成变量、接口、状态、约束、目标函数或可验证规范。
- **control_orientation**：更倾向直接操纵通过测量误差、闭环调节和稳定机制维持目标状态的每个环节，还是通过机制、反馈和边界条件实现间接控制。
- **mission_vs_relational_cost**：为了推进通过测量误差、闭环调节和稳定机制维持目标状态相关目标，是否愿意承受协作摩擦、组织阻力与人际成本。
- **risk_posture**：面对通过测量误差、闭环调节和稳定机制维持目标状态中的不确定性、失败模式与尾部风险，是激进试错、分层防御还是保守回避。
- **relation_to_hierarchy_power**：在涉及通过测量误差、闭环调节和稳定机制维持目标状态的资源、接口、权限和责任边界时，如何理解权威、分工与控制权。
- **conflict_strategy**：当围绕通过测量误差、闭环调节和稳定机制维持目标状态的定义、指标、方案或责任发生分歧时，是诉诸实验、形式化、谈判还是权力推动。
- **time_horizon**：在通过测量误差、闭环调节和稳定机制维持目标状态上更追求短期可见收益，还是愿意为长期能力建设、平台化积累与可维护性投资。
- **rebuild_over_patch**：当通过测量误差、闭环调节和稳定机制维持目标状态暴露出系统性缺陷时，是局部修补还是重写关键假设、接口和架构。
- **metacognition**：是否能反思自己在通过测量误差、闭环调节和稳定机制维持目标状态上的常见盲点，例如迷信指标、忽视边界条件、误把历史成功当普遍规律。

## 5. 典型发现模式

- 开环命令型：下完指令就默认系统会执行。
- 闭环校准型：持续测量并修正，系统更稳。
- 过度控制型：反馈太强，导致震荡。
- 稳态经营型：擅长用小调整保持长期稳定。
