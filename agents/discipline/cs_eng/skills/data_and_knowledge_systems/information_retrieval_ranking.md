# 信息检索与排序分析框架 (Information Retrieval & Ranking)

## 1. 核心理论框架

检索不是存储的附属问题，而是决策效率的核心。这个框架适合观察一个人是否懂得在信息过载中构造好的 query、召回正确候选、再用合适信号排序；也适合分析其在组织中如何建立“找得到真正有用信息”的能力。

## 2. 锚点人物与流派

- **Gerard Salton**：现代信息检索之父。
- **Karen Spärck Jones**：逆文档频率等检索基础思想的重要提出者。
- **Christopher Manning**：将 IR 与 NLP 的方法论系统化。

## 3. 分析方法论

- 面对信息爆炸时，是否先考虑检索策略，而不是盲目通读。
- 是否清楚“相关性”随任务而变化。
- 是否能把召回与精度的张力讲清楚。
- 是否会建设索引、标签和知识入口，而不是只追求内容增量。

## 4. Construct 分析指南

- **agency_orientation**：是否试图通过如何从海量信息中召回、筛选并排序最相关内容主动改写问题结构，而不是只在既有条件下被动应对。
- **epistemic_style**：其判断更依赖关于如何从海量信息中召回、筛选并排序最相关内容的实验、度量、模型与证据，还是经验直觉和少量轶事。
- **abstraction_formalization**：是否能把如何从海量信息中召回、筛选并排序最相关内容抽象成变量、接口、状态、约束、目标函数或可验证规范。
- **control_orientation**：更倾向直接操纵如何从海量信息中召回、筛选并排序最相关内容的每个环节，还是通过机制、反馈和边界条件实现间接控制。
- **mission_vs_relational_cost**：为了推进如何从海量信息中召回、筛选并排序最相关内容相关目标，是否愿意承受协作摩擦、组织阻力与人际成本。
- **risk_posture**：面对如何从海量信息中召回、筛选并排序最相关内容中的不确定性、失败模式与尾部风险，是激进试错、分层防御还是保守回避。
- **relation_to_hierarchy_power**：在涉及如何从海量信息中召回、筛选并排序最相关内容的资源、接口、权限和责任边界时，如何理解权威、分工与控制权。
- **conflict_strategy**：当围绕如何从海量信息中召回、筛选并排序最相关内容的定义、指标、方案或责任发生分歧时，是诉诸实验、形式化、谈判还是权力推动。
- **time_horizon**：在如何从海量信息中召回、筛选并排序最相关内容上更追求短期可见收益，还是愿意为长期能力建设、平台化积累与可维护性投资。
- **rebuild_over_patch**：当如何从海量信息中召回、筛选并排序最相关内容暴露出系统性缺陷时，是局部修补还是重写关键假设、接口和架构。
- **metacognition**：是否能反思自己在如何从海量信息中召回、筛选并排序最相关内容上的常见盲点，例如迷信指标、忽视边界条件、误把历史成功当普遍规律。

## 5. 典型发现模式

- 搜索高手型：能快速定位关键证据，明显提升组织判断速度。
- 收藏癖型：囤积资料但缺乏检索机制。
- 排序直觉型：有好的 relevance sense，但难以形式化复用。
- 入口建设型：重视目录、索引和 query interface，利于团队协作。
