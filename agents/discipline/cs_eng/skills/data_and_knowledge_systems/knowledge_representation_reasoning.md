# 知识表示与推理分析框架 (Knowledge Representation & Reasoning)

## 1. 核心理论框架

知识表示关注的不只是有没有知识，而是知识能否以可组合、可追溯、可推理的方式存在。用于人物分析时，这个框架能看出一个人是否会自发构建 ontology、因果图或规则网络，还是只停留在零散叙事层面。

## 2. 锚点人物与流派

- **John McCarthy**：AI 早期知识表示与形式化常识推理先驱。
- **Judea Pearl**：因果图与结构化知识表示的重要代表。
- **Allen Newell**：统一认知架构与符号系统假说的核心人物。

## 3. 分析方法论

- 是否倾向于把经验整理成实体、关系与规则，而不是堆成故事。
- 是否在推理时区分相关、因果和约束。
- 是否会主动补齐缺失节点与关系，而不是只处理表面事件。
- 是否能让知识被他人继承，而不是只存在于个人脑内。

## 4. Construct 分析指南

- **agency_orientation**：是否试图通过如何把世界知识组织成实体、关系、规则与可推理结构主动改写问题结构，而不是只在既有条件下被动应对。
- **epistemic_style**：其判断更依赖关于如何把世界知识组织成实体、关系、规则与可推理结构的实验、度量、模型与证据，还是经验直觉和少量轶事。
- **abstraction_formalization**：是否能把如何把世界知识组织成实体、关系、规则与可推理结构抽象成变量、接口、状态、约束、目标函数或可验证规范。
- **control_orientation**：更倾向直接操纵如何把世界知识组织成实体、关系、规则与可推理结构的每个环节，还是通过机制、反馈和边界条件实现间接控制。
- **mission_vs_relational_cost**：为了推进如何把世界知识组织成实体、关系、规则与可推理结构相关目标，是否愿意承受协作摩擦、组织阻力与人际成本。
- **risk_posture**：面对如何把世界知识组织成实体、关系、规则与可推理结构中的不确定性、失败模式与尾部风险，是激进试错、分层防御还是保守回避。
- **relation_to_hierarchy_power**：在涉及如何把世界知识组织成实体、关系、规则与可推理结构的资源、接口、权限和责任边界时，如何理解权威、分工与控制权。
- **conflict_strategy**：当围绕如何把世界知识组织成实体、关系、规则与可推理结构的定义、指标、方案或责任发生分歧时，是诉诸实验、形式化、谈判还是权力推动。
- **time_horizon**：在如何把世界知识组织成实体、关系、规则与可推理结构上更追求短期可见收益，还是愿意为长期能力建设、平台化积累与可维护性投资。
- **rebuild_over_patch**：当如何把世界知识组织成实体、关系、规则与可推理结构暴露出系统性缺陷时，是局部修补还是重写关键假设、接口和架构。
- **metacognition**：是否能反思自己在如何把世界知识组织成实体、关系、规则与可推理结构上的常见盲点，例如迷信指标、忽视边界条件、误把历史成功当普遍规律。

## 5. 典型发现模式

- 叙事堆砌型：案例丰富，但难以形成可复用知识结构。
- 本体工程型：天然会设计概念图谱和关系网。
- 规则僵化型：过度依赖显式规则，难以处理模糊现实。
- 因果敏感型：擅长从事件背后提炼机制。
