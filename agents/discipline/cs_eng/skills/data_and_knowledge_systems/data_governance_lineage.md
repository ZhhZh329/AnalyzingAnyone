# 数据治理与谱系分析框架 (Data Governance & Lineage)

## 1. 核心理论框架

数据治理的核心不是管控本身，而是确保信息在流动中不失真、不失责、不断链。这个框架特别适合分析一个人是否关心 source of truth、数据血缘、权限边界与责任归属，以及其对“数据好看但来路不明”的警惕程度。

## 2. 锚点人物与流派

- **W. Edwards Deming**：质量与过程控制思想为数据治理提供基础。
- **D.J. Patil**：推动数据科学工程化与治理实践。
- **Zhamak Dehghani**：在 data mesh 语境下强调数据产品与责任边界。

## 3. 分析方法论

- 是否会追问数据从哪里来、经谁加工、在哪一步可能被污染。
- 是否重视权限、所有权和审计链条。
- 是否能在速度与治理之间找到现实平衡。
- 是否把数据质量问题视为系统问题，而不只怪罪个体。

## 4. Construct 分析指南

- **agency_orientation**：是否试图通过数据来源、加工链、权限、责任与可追溯性主动改写问题结构，而不是只在既有条件下被动应对。
- **epistemic_style**：其判断更依赖关于数据来源、加工链、权限、责任与可追溯性的实验、度量、模型与证据，还是经验直觉和少量轶事。
- **abstraction_formalization**：是否能把数据来源、加工链、权限、责任与可追溯性抽象成变量、接口、状态、约束、目标函数或可验证规范。
- **control_orientation**：更倾向直接操纵数据来源、加工链、权限、责任与可追溯性的每个环节，还是通过机制、反馈和边界条件实现间接控制。
- **mission_vs_relational_cost**：为了推进数据来源、加工链、权限、责任与可追溯性相关目标，是否愿意承受协作摩擦、组织阻力与人际成本。
- **risk_posture**：面对数据来源、加工链、权限、责任与可追溯性中的不确定性、失败模式与尾部风险，是激进试错、分层防御还是保守回避。
- **relation_to_hierarchy_power**：在涉及数据来源、加工链、权限、责任与可追溯性的资源、接口、权限和责任边界时，如何理解权威、分工与控制权。
- **conflict_strategy**：当围绕数据来源、加工链、权限、责任与可追溯性的定义、指标、方案或责任发生分歧时，是诉诸实验、形式化、谈判还是权力推动。
- **time_horizon**：在数据来源、加工链、权限、责任与可追溯性上更追求短期可见收益，还是愿意为长期能力建设、平台化积累与可维护性投资。
- **rebuild_over_patch**：当数据来源、加工链、权限、责任与可追溯性暴露出系统性缺陷时，是局部修补还是重写关键假设、接口和架构。
- **metacognition**：是否能反思自己在数据来源、加工链、权限、责任与可追溯性上的常见盲点，例如迷信指标、忽视边界条件、误把历史成功当普遍规律。

## 5. 典型发现模式

- 结果先行型：先看报表再问来路，容易被假稳定误导。
- 谱系敏感型：对来源与转换链条高度警惕，适合关键系统。
- 治理官僚型：流程完整但灵活性差。
- 责任清晰型：擅长把数据产品化并明确 owner。
