# 平台与生态系统设计分析框架 (Platform & Ecosystem Design)

## 1. 核心理论框架

平台设计不只是建一个产品，而是设计一套让第三方持续加入、协作和受约束的生态规则。用于人物分析时，这个框架特别适合看一个人是产品操盘手，还是生态系统设计者。

## 2. 锚点人物与流派

- **Geoffrey Parker**：平台战略研究的重要代表。
- **Annabelle Gawer**：平台生态与产业组织研究的重要学者。
- **Jean Tirole**：多边市场与平台激励结构的关键理论人物。

## 3. 分析方法论

- 是否从单点产品思维提升到生态位思维。
- 是否理解开放接口与治理约束必须同时存在。
- 是否敏感于网络效应、路径依赖与锁定机制。
- 是否能平衡平台增长与平台秩序。

## 4. Construct 分析指南

- **agency_orientation**：是否试图通过多边平台、开放接口、网络效应与生态治理主动改写问题结构，而不是只在既有条件下被动应对。
- **epistemic_style**：其判断更依赖关于多边平台、开放接口、网络效应与生态治理的实验、度量、模型与证据，还是经验直觉和少量轶事。
- **abstraction_formalization**：是否能把多边平台、开放接口、网络效应与生态治理抽象成变量、接口、状态、约束、目标函数或可验证规范。
- **control_orientation**：更倾向直接操纵多边平台、开放接口、网络效应与生态治理的每个环节，还是通过机制、反馈和边界条件实现间接控制。
- **mission_vs_relational_cost**：为了推进多边平台、开放接口、网络效应与生态治理相关目标，是否愿意承受协作摩擦、组织阻力与人际成本。
- **risk_posture**：面对多边平台、开放接口、网络效应与生态治理中的不确定性、失败模式与尾部风险，是激进试错、分层防御还是保守回避。
- **relation_to_hierarchy_power**：在涉及多边平台、开放接口、网络效应与生态治理的资源、接口、权限和责任边界时，如何理解权威、分工与控制权。
- **conflict_strategy**：当围绕多边平台、开放接口、网络效应与生态治理的定义、指标、方案或责任发生分歧时，是诉诸实验、形式化、谈判还是权力推动。
- **time_horizon**：在多边平台、开放接口、网络效应与生态治理上更追求短期可见收益，还是愿意为长期能力建设、平台化积累与可维护性投资。
- **rebuild_over_patch**：当多边平台、开放接口、网络效应与生态治理暴露出系统性缺陷时，是局部修补还是重写关键假设、接口和架构。
- **metacognition**：是否能反思自己在多边平台、开放接口、网络效应与生态治理上的常见盲点，例如迷信指标、忽视边界条件、误把历史成功当普遍规律。

## 5. 典型发现模式

- 产品中心型：擅长做单一产品，不擅长管理外部生态。
- 生态编排型：能够设计多方共存的价值网络。
- 开放失控型：接口放开过快，治理跟不上。
- 平台统治型：过度控制生态，抑制外部创新。
