# 流量与拥塞控制分析框架 (Traffic & Congestion Control)

## 1. 核心理论框架

拥塞控制体现的是一种动态平衡观：不是一味加速，而是在共享环境中避免系统进入崩溃区。用于人物分析时，它能揭示一个人对流量、需求和组织压力的调节能力。

## 2. 锚点人物与流派

- **Van Jacobson**：TCP 拥塞控制的关键人物。
- **Sally Floyd**：网络流量公平性与拥塞控制研究代表。
- **Raj Jain**：排队与拥塞分析的重要学者。

## 3. 分析方法论

- 是否知道什么时候该放行，什么时候该限流。
- 是否对拥塞前兆和队列积压敏感。
- 是否能在效率与公平之间明确取舍。
- 是否理解过载时局部最优会伤害整体。

## 4. Construct 分析指南

- **agency_orientation**：是否试图通过在负载波动与共享通道中管理流量、延迟与公平性主动改写问题结构，而不是只在既有条件下被动应对。
- **epistemic_style**：其判断更依赖关于在负载波动与共享通道中管理流量、延迟与公平性的实验、度量、模型与证据，还是经验直觉和少量轶事。
- **abstraction_formalization**：是否能把在负载波动与共享通道中管理流量、延迟与公平性抽象成变量、接口、状态、约束、目标函数或可验证规范。
- **control_orientation**：更倾向直接操纵在负载波动与共享通道中管理流量、延迟与公平性的每个环节，还是通过机制、反馈和边界条件实现间接控制。
- **mission_vs_relational_cost**：为了推进在负载波动与共享通道中管理流量、延迟与公平性相关目标，是否愿意承受协作摩擦、组织阻力与人际成本。
- **risk_posture**：面对在负载波动与共享通道中管理流量、延迟与公平性中的不确定性、失败模式与尾部风险，是激进试错、分层防御还是保守回避。
- **relation_to_hierarchy_power**：在涉及在负载波动与共享通道中管理流量、延迟与公平性的资源、接口、权限和责任边界时，如何理解权威、分工与控制权。
- **conflict_strategy**：当围绕在负载波动与共享通道中管理流量、延迟与公平性的定义、指标、方案或责任发生分歧时，是诉诸实验、形式化、谈判还是权力推动。
- **time_horizon**：在在负载波动与共享通道中管理流量、延迟与公平性上更追求短期可见收益，还是愿意为长期能力建设、平台化积累与可维护性投资。
- **rebuild_over_patch**：当在负载波动与共享通道中管理流量、延迟与公平性暴露出系统性缺陷时，是局部修补还是重写关键假设、接口和架构。
- **metacognition**：是否能反思自己在在负载波动与共享通道中管理流量、延迟与公平性上的常见盲点，例如迷信指标、忽视边界条件、误把历史成功当普遍规律。

## 5. 典型发现模式

- 过载冒进型：在系统逼近极限时仍盲目加压。
- 节流稳态型：懂得通过节奏控制保持长期吞吐。
- 公平主义型：强求平均，可能牺牲总效率。
- 瓶颈调参型：善于通过少数关键参数稳定整体。
