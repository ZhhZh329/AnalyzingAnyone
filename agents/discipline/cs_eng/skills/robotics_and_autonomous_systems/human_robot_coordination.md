# 人机体协同分析框架 (Human-Robot Coordination)

## 1. 核心理论框架

人机体协同的难点不在于机器人会不会动，而在于机器人能否以人能理解、能预测、能信任的方式行动。用于人物分析时，这个框架可观察一个人是否善于在共享任务中设计可预期协作。

## 2. 锚点人物与流派

- **Cynthia Breazeal**：社会机器人与人机互动的重要代表。
- **Julie Shah**：人机协作与共享自治研究的重要学者。
- **Maja Mataric**：协作机器人与社会型机器人领域的重要人物。

## 3. 分析方法论

- 是否重视可预测行为与共享心智模型。
- 是否会设计协作中断、接管与恢复机制。
- 是否敏感于人类的安全感和可理解性。
- 是否能在自动化与协商之间找到平衡。

## 4. Construct 分析指南

- **agency_orientation**：是否试图通过机器人与人类在共享空间、共享任务中的协调与安全协作主动改写问题结构，而不是只在既有条件下被动应对。
- **epistemic_style**：其判断更依赖关于机器人与人类在共享空间、共享任务中的协调与安全协作的实验、度量、模型与证据，还是经验直觉和少量轶事。
- **abstraction_formalization**：是否能把机器人与人类在共享空间、共享任务中的协调与安全协作抽象成变量、接口、状态、约束、目标函数或可验证规范。
- **control_orientation**：更倾向直接操纵机器人与人类在共享空间、共享任务中的协调与安全协作的每个环节，还是通过机制、反馈和边界条件实现间接控制。
- **mission_vs_relational_cost**：为了推进机器人与人类在共享空间、共享任务中的协调与安全协作相关目标，是否愿意承受协作摩擦、组织阻力与人际成本。
- **risk_posture**：面对机器人与人类在共享空间、共享任务中的协调与安全协作中的不确定性、失败模式与尾部风险，是激进试错、分层防御还是保守回避。
- **relation_to_hierarchy_power**：在涉及机器人与人类在共享空间、共享任务中的协调与安全协作的资源、接口、权限和责任边界时，如何理解权威、分工与控制权。
- **conflict_strategy**：当围绕机器人与人类在共享空间、共享任务中的协调与安全协作的定义、指标、方案或责任发生分歧时，是诉诸实验、形式化、谈判还是权力推动。
- **time_horizon**：在机器人与人类在共享空间、共享任务中的协调与安全协作上更追求短期可见收益，还是愿意为长期能力建设、平台化积累与可维护性投资。
- **rebuild_over_patch**：当机器人与人类在共享空间、共享任务中的协调与安全协作暴露出系统性缺陷时，是局部修补还是重写关键假设、接口和架构。
- **metacognition**：是否能反思自己在机器人与人类在共享空间、共享任务中的协调与安全协作上的常见盲点，例如迷信指标、忽视边界条件、误把历史成功当普遍规律。

## 5. 典型发现模式

- 机械效率型：只重效率，忽略人类协作体验。
- 共享自治型：擅长让人和系统各司其职。
- 接管混乱型：一旦切换控制权就容易出错。
- 可预测协作型：能让复杂自动系统表现得像可靠队友。
