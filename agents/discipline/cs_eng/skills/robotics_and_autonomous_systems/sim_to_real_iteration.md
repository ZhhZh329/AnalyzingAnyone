# 仿真到现实迭代分析框架 (Sim-to-Real Iteration)

## 1. 核心理论框架

仿真到现实的问题揭示了一个普遍事实：世界总比模型脏。用于人物分析时，这个框架适合看一个人是否懂得利用模拟、沙盒和预演降低成本，同时又尊重真实世界会给模型带来的偏差与破坏。

## 2. 锚点人物与流派

- **Pieter Abbeel**：机器人学习与 sim-to-real 方向的重要推动者。
- **Sergey Levine**：深度强化学习与真实机器人训练的重要代表。
- **Chelsea Finn**：适应性学习与现实迁移研究的代表。

## 3. 分析方法论

- 是否会先在低成本环境中验证策略。
- 是否清楚仿真假设在哪些地方会失效。
- 是否擅长把现实反馈重新注入模型。
- 是否能设计逐步放大风险的验证路线。

## 4. Construct 分析指南

- **agency_orientation**：是否试图通过在仿真环境与真实部署之间建立可迁移的学习和验证循环主动改写问题结构，而不是只在既有条件下被动应对。
- **epistemic_style**：其判断更依赖关于在仿真环境与真实部署之间建立可迁移的学习和验证循环的实验、度量、模型与证据，还是经验直觉和少量轶事。
- **abstraction_formalization**：是否能把在仿真环境与真实部署之间建立可迁移的学习和验证循环抽象成变量、接口、状态、约束、目标函数或可验证规范。
- **control_orientation**：更倾向直接操纵在仿真环境与真实部署之间建立可迁移的学习和验证循环的每个环节，还是通过机制、反馈和边界条件实现间接控制。
- **mission_vs_relational_cost**：为了推进在仿真环境与真实部署之间建立可迁移的学习和验证循环相关目标，是否愿意承受协作摩擦、组织阻力与人际成本。
- **risk_posture**：面对在仿真环境与真实部署之间建立可迁移的学习和验证循环中的不确定性、失败模式与尾部风险，是激进试错、分层防御还是保守回避。
- **relation_to_hierarchy_power**：在涉及在仿真环境与真实部署之间建立可迁移的学习和验证循环的资源、接口、权限和责任边界时，如何理解权威、分工与控制权。
- **conflict_strategy**：当围绕在仿真环境与真实部署之间建立可迁移的学习和验证循环的定义、指标、方案或责任发生分歧时，是诉诸实验、形式化、谈判还是权力推动。
- **time_horizon**：在在仿真环境与真实部署之间建立可迁移的学习和验证循环上更追求短期可见收益，还是愿意为长期能力建设、平台化积累与可维护性投资。
- **rebuild_over_patch**：当在仿真环境与真实部署之间建立可迁移的学习和验证循环暴露出系统性缺陷时，是局部修补还是重写关键假设、接口和架构。
- **metacognition**：是否能反思自己在在仿真环境与真实部署之间建立可迁移的学习和验证循环上的常见盲点，例如迷信指标、忽视边界条件、误把历史成功当普遍规律。

## 5. 典型发现模式

- 纸上完美型：仿真里漂亮，现实里脆弱。
- 迭代迁移型：能用模拟加速学习，同时持续修正现实偏差。
- 现实崇拜型：拒绝模拟，导致试错成本过高。
- 闭环优化型：擅长让仿真与现实形成持续反馈。
